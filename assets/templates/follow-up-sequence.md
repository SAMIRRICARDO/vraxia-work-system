# Follow-up Sequence — VRASHOWS Enterprise
# Sequência completa de relacionamento pós cold outreach

**Versão:** 1.0 — 2026-05-20
**Aplicação:** Empresas contactadas sem resposta ao cold outreach
**Sequência:** D+3 · D+7 · D+15
**Tom:** Executivo · Premium · Consultivo · Relacionamento de longo prazo
**Regra global:** Nunca mais de 3 follow-ups por contato no mesmo ciclo

---

## SEQUÊNCIA COMPLETA — VISÃO GERAL

```
D+0   Cold outreach + Media Kit PDF
D+3   Follow-up leve — reabrir sem repetir o pitch
D+7   Follow-up com credibilidade — case + contexto operacional
D+15  Reabertura elegante — ciclo longo, sem pressão
D+30  Pausa total — silêncio estratégico (não enviar nada)
```

# D+3 — FOLLOW-UP LEVE

**Objetivo:** Reabrir com leveza. Sem repetir o pitch. Lembrete humano.
**Volume:** ~50 palavras
**Attachment:** Não reenviar PDF

**Objetivo:** Agregar valor com case concreto. Mostrar resultado, não apenas proposta.
**Volume:** ~90 palavras
**Attachment:** Opcional — só se o contato não abriu o primeiro email

# REGRAS GLOBAIS DA SEQUÊNCIA

1. **Nunca enviar mais de 3 emails no mesmo ciclo** por contato
2. **Nunca repetir o mesmo subject** em emails diferentes
3. **Nunca reutilizar bullets** do cold outreach
4. **Sempre personalizar** pelo menos uma linha por segmento
5. **Nunca:** "última chance", "urgente", "expira em X dias"
6. **Nunca:** reenviar PDF sem ser solicitado
7. **Sempre:** espaçamento real entre envios (D+3, D+7, D+15)
8. **Após D+15:** silêncio de 30 dias mínimo
9. **Tom:** par-a-par executivo — nunca vendedor falando com cliente
10. **CTA:** sempre consultivo ou de abertura — nunca transacional

# INTEGRAÇÃO COM O RUNTIME

Para gerar follow-ups via agente:

```bash
# D+3
tsx scripts/run-outbound-batch.ts \
  --queue data/outreach/outbound-follow-d3.json \
  --template follow-up-d3 \
  --live --limit 5

# D+7
tsx scripts/run-outbound-batch.ts \
  --queue data/outreach/outbound-follow-d7.json \
  --template follow-up-d7 \
  --live --limit 5

# D+15
tsx scripts/run-outbound-batch.ts \
  --queue data/outreach/outbound-follow-d15.json \
  --template follow-up-d15 \
  --live --limit 5
```

Campos obrigatórios no queue entry para follow-ups:

```json
{
  "followUpStage": "d3" | "d7" | "d15",
  "originalSentAt": "2026-05-19T19:28:41.932Z",
  "originalSubject": "Como reduzir ruído operacional...",
  "segment": "telecom" | "cloud" | "fintech" | "ai-sec" | "marketing"
}
```
